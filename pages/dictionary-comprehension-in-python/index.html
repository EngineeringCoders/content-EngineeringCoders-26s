What is Dictionary Comprehension?

Dictionary comprehension is a concise and powerful way to create dictionaries using:

  • loops
  • conditions
  • transformations
  • filtering

Instead of writing multiple lines using traditional loops, Python allows dictionary creation in a compact readable format.

Traditional Dictionary Creation

Python
12345678
numbers = [1, 2, 3, 4]

squares = {}

for num in numbers:
    squares[num] = num * num

print(squares)

Output

Python
1
{1: 1, 2: 4, 3: 9, 4: 16}

Dictionary Comprehension Version

Python
12345678
numbers = [1, 2, 3, 4]

squares = {
    num: num * num
    for num in numbers
}

print(squares)

How Python Stores Dictionaries Internally

Python dictionaries use:

  • Hash tables
  • Hash functions
  • Buckets
  • Key references
  • Value references

This allows:

text
1
Very fast lookups → Average O(1)

Time Complexity

Dictionary Creation

Space Complexity

Real Memory Discussion 🧠

Dictionaries consume more memory than lists.

Why?

Because dictionaries store:

  • keys
  • values
  • hash values
  • bucket metadata
  • collision handling structures

List vs Dictionary Memory

List

text
1
[1][2][3][4]

Compact memory layout.

Dictionary

text
123
Hash Buckets

Key -> Hash -> Value

Faster lookup but more RAM usage.

Common Beginner Mistakes 🚨

Mistake 1: Complex Logic Inside Comprehension

Bad:

Python
12345
{
    x: complicated_function(x)
    for x in data
    if multiple_conditions(x)
}

Hard to debug and understand.

Better Approach

Use normal loops if logic becomes complicated.

Readable code is production-grade code.

Mistake 2: Using Expensive Operations Inside Comprehension

Bad Example

Python
123456
words = ["apple", "banana", "apple"]

freq = {
    word: words.count(word)
    for word in words
}

Why Bad?

Python
1
words.count(word)

scans the list every time.

Complexity becomes:

text
1
O(n²)

Better Approach

Python
123456789
freq = {}

for word in words:

    if word in freq:
        freq[word] += 1

    else:
        freq[word] = 1

Efficient:

text
1
O(n)

Generator vs Dictionary Comprehension

Important distinction:

Dictionary Comprehension

Python
1
{x:x*x for x in range(1000000)}

Creates everything immediately.

Consumes large memory.

Generator Expression

Python
1
(x*x for x in range(1000000))

Produces values lazily.

Tiny memory footprint.

Final Takeaways

Use Dictionary Comprehension When:

  • transformation is simple
  • readability remains clean
  • single-pass creation needed
  • lookup performance matters

Avoid When:

  • nested logic becomes messy
  • debugging becomes difficult
  • memory usage is critical
  • operations become computationally expensive

Golden Rule ✨

Dictionary comprehensions are not just shorter syntax.

They are:

  • data transformation tools
  • lookup table generators
  • high-performance structures
  • foundational building blocks for scalable backend systems

Once understood deeply, they become one of Python’s most powerful weapons ⚡
